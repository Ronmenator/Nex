Standard Library Overview

Nex ships with a built-in standard library of 15 modules covering math, strings, I/O, collections, networking, threading, and more. These are always available without any project configuration.

Importing Modules

All standard library modules live under the std namespace. Import an entire module or pick individual functions:

// Import the whole module
import std.math

// Import specific functions
from std.math import sqrt, pow

// Alias a module
import std.collections as col
Note

std.core is automatically imported in every Nex program. You never need to explicitly import print, println, or Object.

Looking for crypto, http, regex, torch, or UI?

These have been moved to standalone Standard Libraries. Add them to your project.toml and import by library name (e.g. import crypto.hash, import http.client).

Module Index

std.core

print, println, Object — always available without imports.

std.runtime

Disposable, gc_collect — runtime interfaces and garbage collection control.

std.math

abs, sin, cos, sqrt, pow, random, floor, ceil, round, log, exp, min, max, clamp

std.string

trim, split, replace, upper/lower, starts_with, ends_with, contains, index_of, reverse

std.convert

parse_int, parse_float, parse_bool, char_to_str — type conversion and parsing.

std.collections

List, Map, Set with sort, reverse, clear, contains — generic collections.

std.io

File, read_line, file_exists, file_delete, file_copy, mkdir, list_dir

std.env

env_get, env_set, env_has, args, cwd — environment variables and process info.

std.time

now_millis, now_nanos, sleep_millis, elapsed_millis — time measurement and delays.

std.path

join, parent, file_name, extension, stem, is_absolute, normalize, separator

std.json

parse, stringify, get_string, get_int, get_float, get_bool — JSON handling.

std.process

exec, exec_output, exit, pid, spawn, wait — process management.

std.net

TCP: connect, listen, accept, send, recv, close
UDP: bind, send, recv, close

std.threading

spawn, join, sleep, current_id, mutex: new, lock, unlock, free

std.logging

debug, info, warn, error, set_level, with_tag — structured logging.

std.testing

assert, assert_eq, assert_ne, assert_true — unit test assertions.
