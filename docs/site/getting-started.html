Installation & Setup

Get the Nex toolchain running on your machine in under five minutes.

Quick Install (Windows)

Install Nex with a single PowerShell command:

irm https://raw.githubusercontent.com/Ronmenator/Nex/master/install.ps1 | iex

This downloads the latest release from GitHub, extracts it to ~/.nex/, and adds ~/.nex/bin to your PATH. All standard libraries and their native DLLs are included.

After Install

Restart your terminal after installation, then verify with nex --help.

Prerequisites

Nex's JIT mode requires no external dependencies. For AOT compilation (producing standalone executables), you need a system linker:

PlatformLinker
WindowsMSVC Build Tools (link.exe), GCC (MinGW), or Clang
LinuxGCC or Clang (usually pre-installed)
macOSXcode Command Line Tools (GCC/Clang)

Directory Layout

After building, the toolchain is installed under the nex/ directory:

nex/
├── bin/
│   ├── nex.exe          // Main CLI (build, run, fmt, lint, repl)
│   ├── nexc.exe         // Standalone compiler frontend
│   └── nex-lsp.exe      // Language server (LSP)
├── config/
│   ├── nex.toml         // Toolchain configuration
│   └── project.toml     // Template for new projects
└── examples/
    ├── hello/
    └── console_showcase/

Add to PATH

Windows (PowerShell)

# Current session only
$env:PATH += ";C:\path\to\nex\bin"

# Permanent: add via System Environment Variables dialog

Linux / macOS

# Add to ~/.bashrc or ~/.zshrc
export PATH="$PATH:/path/to/nex/bin"

Verify Installation

nex --help

You should see the available commands:

Nex compiler toolchain

Usage: nex <command> [args]

Commands:
  new <name> [--lib]    Create a new project
  build [path]          Compile to a native executable (AOT)
  run [path] [-- args]  JIT compile and execute
  <file>.nex [args]      Shorthand for: nex run <file>.nex
  install               Build release binaries
  fmt <file>            Format source code
  lint <file>           Run linter
  repl                  Interactive REPL
  clean                 Remove build artifacts

Create Your First Project

nex new my_project
cd my_project

This creates a project with the following structure:

my_project/
├── project.toml
└── src/
    └── main.nex

Build & Run

Tip

Use nex run for fast iteration (JIT) and nex build when you need a standalone binary.

# JIT compile and execute immediately
nex run

# Or use the shorthand for a single file
nex src/main.nex

# Compile to native executable
nex build

Execution Modes

JIT Mode (default)

Compiles in memory using Cranelift JIT and executes immediately. No intermediate files, no external tools needed. Feels like running a script.

AOT Mode

Compiles to a native object file via Cranelift, then links with the system linker to produce a standalone executable. Use this for production deployments.
