Interfaces

Contracts that define method signatures for classes and structs to implement.

Declaring an Interface

public interface Greeter {
    def greet(name: String) -> String
}

Interface Rules

  • Declare method signatures only — no fields, no state
  • Methods are implicitly public and virtual
  • Can extend other interfaces: interface A : B { }
  • Can have type parameters: interface Comparable[T] { }
  • Cannot be instantiated directly

Implementing in a Class

Classes implement interfaces by listing them after the : in the class declaration, alongside any base classes:

interface Greeter {
    def greet(name: String) -> String
}

class FriendlyGreeter : Greeter {
    def greet(name: String) -> String {
        return "Hey there, " + name + "!"
    }
}

Implementing in a Struct

interface Printable {
    def to_string() -> String
}

struct Color : Printable {
    r: Int
    g: Int
    b: Int

    def to_string() -> String {
        return "rgb(" + self.r + "," + self.g + "," + self.b + ")"
    }
}

Class with Base and Interface

class ConsoleGreeter : BaseGreeter, Greeter {
    def greet(name: String) -> String {
        return self.Base::format(name)
    }
}

Interface Inheritance

interface Serializable {
    def serialize() -> String
}

interface Storable : Serializable {
    def save(path: String) -> Unit
}

// Must implement both serialize() and save()
class Document : Storable {
    def serialize() -> String { return "{}" }
    def save(path: String) -> Unit { }
}

Generic Interfaces

interface Comparable[T] {
    def compare_to(other: T) -> Int
}

The Disposable Interface

The built-in Disposable interface is used for deterministic resource cleanup with using blocks:

public interface Disposable {
    def dispose() -> Unit
}

See Resource Management for details on using blocks.
