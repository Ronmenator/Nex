Modules & Imports

Organizing code into modules, importing dependencies, and controlling visibility.

Module System

Each .nex file is a module. The module name is derived from the file's path relative to the src/ directory:

src/main.nex          → main
src/net/http.nex      → net.http
src/utils/helpers.nex → utils.helpers

Import Syntax

Import a Module

import std.math

Import with Alias

import std.math as math

Import Specific Items

from std.io import open_console
from std.collections import List, Map
Not Supported in v1

Wildcard imports (e.g., from std.math import *) are not supported.

Visibility

All declarations are module-internal by default. Use public to export a declaration for use in other modules:

// Only visible within this file/module
def internal_helper() -> Unit { }

// Visible to any module that imports this one
public def api_function() -> Unit { }

// Public class with some internal members
public class Client {
    public url: String     // exported
    token: String          // internal to module
}

Library Dependencies

External libraries are declared in project.toml under the [libs] section. Libraries can come from GitHub, the global cache, or a local path:

[libs]
nex3d = { git = "nexlang/nex3d", version = "0.1.0" }
utils = "1.2.0"
graphics = { path = "../graphics" }

The library name becomes the import namespace prefix:

import nex3d.engine       // imports nex3d/src/engine.nex
import graphics.shapes    // imports graphics/src/shapes.nex

Installing Libraries

Install libraries from GitHub using nex install:

$ nex install user/repo            # latest tag
$ nex install user/repo:0.1.0      # specific version

This downloads the library to the global cache at ~/.nex/libs/ and adds an entry to your project.toml. See Build Tool for full details.

Library Rules

  • A library is a Nex project with its own project.toml and src/ directory
  • Library sources are compiled alongside the main project and linked into a single executable
  • GitHub is the only remote source in v1 (no central registry)
  • v1 uses flat resolution — libraries cannot declare their own [libs] dependencies
  • If a library declares native = "name" in its project.toml, the installer downloads prebuilt DLLs from GitHub release assets

Standard Library Imports

Standard library modules are imported using the std prefix:

import std.math
import std.io
import std.json
import std.collections

The std.core module (print, println, Object) is available without import.

Standalone standard libraries (crypto, http, regex, torch, nex_ui, nex3d) are imported by their library name after adding them to project.toml:

import crypto.hash        // from the crypto library
import http.client         // from the http library
import torch.tensor        // from the torch library

Project Structure

my_project/
├── project.toml       // project configuration
└── src/
    ├── main.nex        // entry point module
    ├── models.nex      // → models
    └── net/
        ├── client.nex  // → net.client
        └── server.nex  // → net.server
