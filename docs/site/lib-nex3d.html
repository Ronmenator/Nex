nex3d

3D Game Engine library for building 2D and 3D games. Provides math types (Vec2, Vec3, Vec4), engine lifecycle (window, run loop), and GPU-accelerated rendering via WGPU.

Build Requirement

For engine functionality, build the Nex toolchain with cargo build --features engine -p nex. The nex3d.math module works without this feature.

Setup

Add nex3d to your project.toml:

[libs]
nex3d = { path = "../libs/nex3d" }

Modules

nex3d.math

Pure Nex vector and matrix types using std.math. No runtime engine dependency.

import nex3d.math

Vec2

2D vector for sprites, UI, and 2D games.

vec2(x: Float, y: Float) -> Vec2
vec2_add, vec2_sub, vec2_mul, vec2_dot, vec2_length, vec2_normalize

Vec3

3D vector for positions, directions, RGB colors.

vec3(x, y, z), vec3_add, vec3_sub, vec3_mul, vec3_dot, vec3_length, vec3_normalize

Vec4

4D vector for homogeneous coordinates and RGBA.

vec4(x, y, z, w), vec4_from_vec3

nex3d.engine

Window creation and game main loop. Requires --features engine.

import nex3d.engine

engine_create_window

def engine_create_window(title: String, width: Int64, height: Int64) -> Int64

Creates a game window. Returns a handle (or 0 if engine feature is disabled).

engine_run

def engine_run(handle: Int64) -> Unit

Runs the event loop. Blocks until the window is closed.

engine_quit / engine_destroy

Signals quit and releases resources.

Example

import nex3d.math
import nex3d.engine

def main() {
    v = vec2(100.0f, 200.0f)
    handle = engine_create_window("My Game", 800, 600)
    if (handle != 0) {
        engine_run(handle)
        engine_destroy(handle)
    }
    return
}
