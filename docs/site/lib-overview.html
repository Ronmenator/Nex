Standard Libraries

Nex ships with a collection of standalone libraries for cryptography, networking, regular expressions, machine learning, UI, and 3D graphics. Unlike the built-in standard library (std.*), these are opt-in — add them to your project.toml and import them by name.

How Libraries Work

Each standard library is a self-contained package in the libs/ directory. A library consists of:

  • project.toml — declares the library name, version, and optional native DLL
  • src/*.nex — Nex source modules that wrap the native functions with a clean API
  • Native DLL (optional) — a compiled shared library (.dll / .so) containing the low-level implementation

Adding a Library to Your Project

Reference the library in your project.toml:

[libs]
crypto = { path = "../libs/crypto" }
http   = { path = "../libs/http" }

Then import individual modules:

import crypto.hash
import http.client

hash = sha256("hello")
var resp = get("https://api.example.com/status")
Standard Library vs Standard Libraries

The Standard Library (std.*) is always available — it provides core functions like math, strings, I/O, collections, and more. Standard Libraries are standalone packages that you add to your project when needed. They ship with Nex but are not loaded unless referenced in project.toml.

Library Index

crypto

SHA-1, SHA-256, SHA-512, MD5, HMAC-SHA256, AES-256-GCM, Ed25519, PBKDF2, Argon2, Base64, random bytes.

http

HTTP client (GET/POST) and server with request routing, JSON/text/HTML responses.

regex

Regular expression compilation, matching, searching, and replacement.

torch

PyTorch integration: tensors, neural network layers, loss functions, optimizers, model I/O, CUDA support, scalar/element-wise math, masking, reductions, convenience builders, and high-level training utilities.

nex_ui

Cross-platform UI: desktop windows (WGPU), terminal TUI, widgets, flexbox layout, canvas drawing, dialogs.

net

TLS/SSL client connections and RFC 6455 WebSocket server with send/recv messaging.

nex3d

3D game engine with 18 modules: vector math, window management, GPU rendering, sprites, 3D models, animation, audio, lighting, fonts, gamepad input, render targets, bounding volumes, and game state management.

Creating Your Own Library

You can create libraries following the same pattern. Create a directory with a project.toml and src/ folder:

my_lib/
  project.toml
  src/
    main.nex
    utils.nex

The project.toml declares your library:

[project]
name = "my_lib"
version = "0.1.0"

# Optional: point to a native DLL for FFI functions
# native = "my_lib_native"

Other projects reference it with a relative path in their [libs] section.
