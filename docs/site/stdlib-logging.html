std.logging

Structured logging with severity levels and tags.

import std.logging

Log Functions

log_debug

def log_debug(msg: String) -> Unit

Logs a message at the DEBUG level (0). Intended for detailed diagnostic information useful during development.

log_info

def log_info(msg: String) -> Unit

Logs a message at the INFO level (1). Suitable for general operational messages that confirm the program is working as expected.

log_warn

def log_warn(msg: String) -> Unit

Logs a message at the WARN level (2). Indicates a potential problem or unexpected situation that doesn't prevent the program from continuing.

log_error

def log_error(msg: String) -> Unit

Logs a message at the ERROR level (3). Indicates a serious problem that may prevent part of the program from functioning correctly.

Configuration

log_set_level

def log_set_level(level: Int) -> Unit

Sets the minimum severity level for log output. Messages below this level are suppressed. Level values:

ValueLevel
0DEBUG
1INFO
2WARN
3ERROR

The default level is 0 (DEBUG), meaning all messages are shown.

Tagged Logging

log_with_tag

def log_with_tag(tag: String, msg: String) -> Unit

Logs a message with a custom tag prefix. Useful for categorizing log output by subsystem or component. The tag is included in the log output as [tag].

Complete Example

import std.logging

// Set minimum log level to INFO (suppress DEBUG)
log_set_level(1)

// These messages demonstrate each severity level
log_debug("Cache hit ratio: 0.95")       // suppressed (below INFO)
log_info("Server started on port 8080")   // [INFO] Server started on port 8080
log_warn("Disk usage above 80%")          // [WARN] Disk usage above 80%
log_error("Failed to connect to database") // [ERROR] Failed to connect to database

// Tagged logging for different subsystems
log_with_tag("HTTP", "Request received: GET /api/users")
log_with_tag("DB", "Query executed in 12ms")
log_with_tag("AUTH", "Token validated for user 42")

// Switch to WARN-only in production
log_set_level(2)
log_info("This will not appear")   // suppressed
log_warn("This will appear")       // [WARN] This will appear
