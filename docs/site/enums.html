Enums

Define types with a fixed set of named variants.

Declaring an Enum

An enum declares a type that can be one of several named variants. Use the enum keyword followed by the name and a list of variants inside braces:

enum Direction {
    North,
    South,
    East,
    West
}

Each variant is a distinct value of the enum type. Variants are separated by commas:

enum Color {
    Red,
    Green,
    Blue
}

Public Enums

By default, enums are private to the module they are defined in. To make an enum visible to other modules, use the public keyword:

public enum Status {
    Active,
    Inactive,
    Pending
}

Other modules that import this module can then use Status and all of its variants.

Using Enum Values

Access a specific variant using dot notation: EnumName.VariantName.

dir = Direction.North
color = Color.Red

println(dir)    // North
println(color)  // Red

Enum values can be stored in variables, passed to functions, and returned from functions just like any other value:

def default_direction() -> Direction {
    return Direction.North
}

def print_direction(d: Direction) -> Unit {
    println(d)
    return
}

d = default_direction()
print_direction(d)  // North

Pattern Matching with Enums

Enums work naturally with match expressions. Each arm matches a specific variant:

def describe(dir: Direction) -> String {
    match dir {
        Direction.North -> return "heading north"
        Direction.South -> return "heading south"
        Direction.East -> return "heading east"
        Direction.West -> return "heading west"
    }
}

You can also use the wildcard pattern _ to match any remaining variants:

def is_vertical(dir: Direction) -> Bool {
    match dir {
        Direction.North -> return true
        Direction.South -> return true
        _ -> return false
    }
}

Attributes on Enums

Enums support attributes using bracket syntax. For example, the [Reflectable] attribute enables runtime reflection on the enum:

[Reflectable]
enum Color {
    Red,
    Green,
    Blue
}

Attributes are placed on the line immediately before the enum declaration.

Comparison

Enum values can be compared using the == operator. Two enum values are equal if they are the same variant:

a = Color.Red
b = Color.Red
c = Color.Blue

println(a == b)  // true
println(a == c)  // false
Simple variants only

Nex v1 enums have simple named variants with no associated data. Each variant is a distinct constant value. Associated data on enum variants (such as Some(value)) is planned for a future release.

Complete Example

Here is a full example that declares two enums, uses pattern matching, and compares values:

enum Direction {
    North,
    South,
    East,
    West
}

enum Color {
    Red,
    Green,
    Blue
}

def direction_name(d: Direction) -> String {
    match d {
        Direction.North -> return "North"
        Direction.South -> return "South"
        Direction.East -> return "East"
        Direction.West -> return "West"
    }
}

def color_code(c: Color) -> Int {
    match c {
        Color.Red -> return 0
        Color.Green -> return 1
        Color.Blue -> return 2
    }
}

def main() -> Unit {
    dir = Direction.East
    println(direction_name(dir))  // East

    c1 = Color.Green
    c2 = Color.Green
    println(c1 == c2)  // true
    println(color_code(c1))  // 1
}
